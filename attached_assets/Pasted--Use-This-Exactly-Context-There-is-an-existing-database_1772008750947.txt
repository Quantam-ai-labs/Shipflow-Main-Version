

## Use This Exactly 

> **Context:**

> There is an existing database and existing Shopify integration logic.
> There is a UI button labeled **“Refresh all data”** in the dashboard header.
>
> **Task:**
> Implement logic so that when the **“Refresh all data”** button is clicked:
>
> 1. **Safely sync ALL relevant Shopify data** (orders, customers, products, inventory, financial stats)
>    from Shopify → into the existing Replit database.
> 2. After sync completes, **refetch and refresh all dashboard sections**, including but not limited to:
>
>    * Order counts
>    * Delivered / Pending / In-Transit stats
>    * COD Pending Collection
>    * Order search results
>    * Any analytics widgets currently visible on the dashboard
> 3. **DO NOT break or refactor existing logic, components, or API routes.**
> 4. **DO NOT delete or overwrite unrelated database data.**
> 5. Sync must be **idempotent**:
>
>    * Existing records should be updated by Shopify ID
>    * New records should be inserted
>    * No duplicate data
> 6. Show a **loading state** on the button while syncing:
>
>    * Disable the button during sync
>    * Re-enable it after completion or failure
> 7. Handle failures safely:
>
>    * If Shopify API fails, database must remain unchanged
>    * Log errors clearly
>
> **Safe Restore Point (MANDATORY):**
>
> * Before making any changes, create a **named safe checkpoint** called:
>
>   ```
>   DASH_SYNC_SAFE_POINT
>   ```
> * This checkpoint must allow reverting all changes related to this task only.
> * Do NOT include unrelated files in this checkpoint.
>
> **Revert Instruction Support:**
>
> * Changes must be structured so I can later say:
>
>   ```
>   revert the Dash changes
>   ```
>
>   and the system can cleanly roll back to `DASH_SYNC_SAFE_POINT`.
>
> **Implementation Notes:**
>
> * Use existing Shopify API credentials and config (do not hardcode new keys).
> * Prefer existing data-fetching utilities or API routes.
> * If a new API endpoint is required, namespace it clearly (e.g. `/api/sync/shopify`).
> * Keep changes minimal, readable, and isolated.
>
> **Final Requirement:**
> After implementation, clicking **“Refresh all data”** must result in the dashboard reflecting **fresh Shopify data**, end-to-end, without breaking anything else.
