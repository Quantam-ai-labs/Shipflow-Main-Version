5) Prevent duplicates + idempotency

Booking must be “safe”:

If the same order is booked twice (double click, refresh, retries), do NOT create a second courier booking.

Implement idempotency:

Before booking, check if order already has courier_tracking_number for that courier. If yes, skip and report “Already booked”.

Also maintain a booking_jobs table keyed by (merchantId, orderId, courier) with states: queued, processing, success, failed. If request repeats while processing, return current job status.

Log every booking attempt with request payload + response.
