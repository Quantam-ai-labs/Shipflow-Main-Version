7) WEBHOOK RECEIVING + DATA CONSISTENCY (CRITICAL)
========================================================
Webhook endpoints must:
A) Verify authenticity:
- Validate Shopify HMAC signature using the raw request body and app secret.
- If invalid -> 401 reject.

B) Idempotency:
- Compute payloadHash = sha256(rawBody + topic + shop)
- If payloadHash already exists -> respond 200 and mark IGNORED_DUPLICATE.

C) Ordered update guard:
- Ensure we do not overwrite newer DB data with older webhook payload.
- Compare payload.updated_at with Order.updatedAtShopify:
  - Only apply update if payload.updated_at >= current stored updatedAtShopify.
  - Otherwise ignore.

D) Hybrid “fetch full order” strategy:
- For orders/updated webhook:
  - Either trust payload if complete OR
  - Fetch the full order from Shopify Admin API using shopifyOrderId for correctness, then upsert.
Choose safest: fetch full order for updated events if your current payload processing is incomplete.

E) Always respond fast (Shopify retries):
- Return 200 quickly.
- If needed, queue background processing (simple in-process queue is ok) but do not block too long.
