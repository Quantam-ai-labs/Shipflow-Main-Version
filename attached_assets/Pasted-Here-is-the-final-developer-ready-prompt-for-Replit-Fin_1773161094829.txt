## Final Replit Developer Prompt

Build a robust **Order Confirmation Automation System** inside our existing ecommerce operations app. Orders already sync automatically from Shopify into the app. This system must handle **WhatsApp confirmation, confirmation pending, robo call automation, manual resolution, queue management, activity logs, notifications, order state locking, and Shopify write-back tags**.

## Core Business Rule

From now on, the app must **not depend on Shopify tags for decision-making**.

The app must run entirely on its **own internal workflow, statuses, timestamps, logs, and decision engine**.

Shopify tags are only a **backup / write-back mirror** so that if something is disturbed in the app, a backup trace still exists in Shopify.

This must be clearly reflected in the implementation.

# 1. Order Entry Flow

When a new order comes from Shopify into the app:

* create/import the order into the app
* place it in **New Orders**
* immediately apply these initial tags:

  * `Whatsapp-Pending`
  * `Pending`

These tags must be applied:

* inside the app
* and written back to Shopify as backup

Immediately after order creation, send a **WhatsApp confirmation message** to the customer.

The WhatsApp message must contain two clear actions:

* **Confirm**
* **Cancel**

# 3. Repeated / Opposite WhatsApp Responses

The system must handle multiple WhatsApp actions safely until the order is **Booked**.

## Case A: Customer confirms, then later cancels

Example:

* order already moved to Ready to Ship
* before booking, customer presses Cancel

Then:

* remove previous tags:

  * `Whatsapp-Confirmed`
  * `Confirm`
* add:

  * `Whatsapp-Cancelled`
  * `Cancel`
* move order to **Hold**
* require manual confirmation
* log that conflicting customer intent was detected

## Case B: Customer cancels, then later confirms

Example:

* order already moved to Cancelled
* before booking, customer presses Confirm

Then:

* remove previous tags:

  * `Whatsapp-Cancelled`
  * `Cancel`
* add:

  * `Whatsapp-Confirmed`
  * `Confirm`
* move order to **Hold**
* require manual confirmation
* log that conflicting customer intent was detected

### Important Rule

If the customer gives **both confirm and cancel responses** before booking, the order must go to **Hold for manual review**.

Do not trust the later message blindly for shipping.
The system should preserve the latest tags but still shift workflow to **Hold** for manual confirmation.

# 4. Global Tag Replacement Rule

The system must **never stack conflicting tags**.

Whenever a new status/source tag is applied:

* all previous active status/source tags must be removed first
* only the new current-state tags should remain

Tags must always reflect the **current state only**.

## Status Tags Group

Only one of these may exist at a time:

* `Pending`
* `Confirm`
* `Cancel`
* `Querry`

## Source Tags Group

Only one relevant source tag may exist at a time:

* `Whatsapp-Pending`
* `Whatsapp-Confirmed`
* `Whatsapp-Cancelled`
* `Whatsapp-Querry`
* `Manual-Confirm`
* `Manual-Cancel`
* `Robo-Confirmed`
* `Robo-Cancelled`

## Required Tag Update Sequence

Every tag update must follow this exact order:

1. identify current active status/source tags
2. remove old active tags
3. apply new tags
4. write back to Shopify
5. log the full tag transition

This logic must be hardcoded and must not be skipped.

# 5. Confirmation Pending Logic

If no valid WhatsApp confirm/cancel response is received within **12 hours** after order creation:

* move order from **New Orders** to **Confirmation Pending**

Orders in **Confirmation Pending** become eligible for robo call only if:

* no valid WhatsApp confirmation/cancellation exists yet
* 12 hours have passed
* current time is within the allowed robo call time window

Before triggering robo call, the system must re-check:

* did WhatsApp response arrive already?
* if yes, do not call
* process WhatsApp result instead

Create a **Robo Call Settings** section where admin can configure:

* allowed call start time
* allowed call end time

Example:

* allowed between 10:00 AM and 8:00 PM

* do not trigger the call immediately
* place the order into a **call queue**
* schedule the call for the next allowed start time
* normally next day at allowed start time if current window has passed

# 10. Order Lock Rule After Booking

Responses from WhatsApp or Robo Call can be accepted only **until the order is Booked** with a courier.

Once an order reaches **Booked**:

* the flow becomes locked
* no further WhatsApp or Robo Call response should change status
* no new tags should be applied based on those late responses
* Shopify tags should not be changed from those late responses
* internal workflow state must remain unchanged

If any response arrives after booking:

* create an alert
* show it in notifications
* keep audit log
* do not entertain it as a live workflow change

This lock must be strict and hardcoded.
