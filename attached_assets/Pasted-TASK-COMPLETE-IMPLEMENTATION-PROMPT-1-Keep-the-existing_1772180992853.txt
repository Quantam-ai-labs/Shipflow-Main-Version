========================================================
PART 5 — EVIDENCE, STAGE, INSIGHTS (SMART BUT DETERMINISTIC)
========================================================

CONFIG CONSTANTS
- MIN_SPEND_FOR_EVIDENCE_PKR = 2000
- MIN_DELIVERED_FOR_EVIDENCE = 10
- DEFAULT_EVAL_WINDOW_HOURS = 48

EVIDENCE-READY RULE
evidence_ready if:
- delivered_count >= 10
OR
- spend_total >= 2000 PKR
If delivered missing, rely on spend.

STAGE LOGIC
- Exploration: no actions yet OR not evidence_ready
- Validation: evidence_ready and not proven scaler
- Scaling: evidence_ready and latest evaluated event shows positive leverage (below)
- Decline: evidence_ready and latest evaluated event shows negative leverage

LEVERAGE INDEX (LI)
LI = net_profit_delta / max(spend_delta, 1)
Interpretation:
- Strong: net_profit_delta > 0 AND LI >= 1.0
- Neutral: net_profit_delta > 0 AND LI between 0 and 1.0
- Weak: net_profit_delta ~ 0 while spend increased
- Negative: net_profit_delta < 0

TREND
- Improving if net_profit_delta > 0 AND (cpa_delta <= 0 if cpa exists)
- Worsening if net_profit_delta < 0
- Flat otherwise

INSIGHT + NEXT MOVE (1 sentence each; short)
If evaluation pending:
Insight: “Collecting data—evaluation pending.”
Next Move: “Hold changes until window completes.”

If strong leverage:
Insight: “Positive leverage—profit rose faster than spend.”
Next Move: “Scale +10–20% and keep variables stable for next window.”

If neutral leverage:
Insight: “Profit improved but scaling efficiency is moderate.”
Next Move: “Hold budget; refine creative/offer before scaling again.”

If weak leverage:
Insight: “Spend increased without meaningful profit lift.”
Next Move: “Stop scaling; fix bottleneck then retest.”

If negative leverage:
Insight: “Change reacted negatively—loss increased after modification.”
Next Move: “Reduce -30% or revert; test one focused change next.”

For campaigns with no actions:
Insight: “New campaign—gathering evidence.”
Next Move: “Wait until evidence threshold, then review.”
