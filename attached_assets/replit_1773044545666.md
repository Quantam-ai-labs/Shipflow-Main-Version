# WAppAutomate - WhatsApp Automation SaaS

## Overview
Production-ready WhatsApp automation SaaS for ecommerce. Sends/receives real WhatsApp messages via Meta Cloud API, integrates with Shopify webhooks (orders, abandoned carts, shipping), runs actual automation workflows, supports full template creation/management, and has real-time updates via WebSocket. Zero demo data — starts clean.

## Architecture
- **Frontend**: React 19 + Vite + Tailwind v4 + shadcn/ui + wouter routing + TanStack Query
- **Backend**: Express 5 on Node.js with TypeScript
- **Database**: PostgreSQL with Drizzle ORM
- **Real-time**: WebSocket server on `/ws` for live message/event updates
- **WhatsApp**: Meta Cloud API (Graph API v21.0) for sending/receiving messages
- **Shopify**: Shopify Admin API 2024-10 with OAuth app installation flow + HMAC webhook verification
- **Styling**: Tailwind CSS v4 with CSS variable theming, Inter + Plus Jakarta Sans fonts

## Database Models
- **users** - Authentication (username/password)
- **contacts** - CRM contacts with tags, source, notes
- **conversations** - Chat threads linked to contacts
- **messages** - Messages with whatsappMessageId, status (sent/delivered/read/failed), errorMessage, metadata (jsonb for button reply info)
- **campaigns** - Campaigns with templateName, audienceTags, sentCount/deliveredCount/readCount/failedCount
- **automations** - Workflow automations with triggerType and config (messageText, delayMinutes, templateName)
- **connection_settings** - Meta/WhatsApp API credentials, displayPhoneNumber, verifiedName, qualityRating
- **templates** - WhatsApp message templates synced from Meta (metaId, name, language, category, status, components)
- **shopify_settings** - Shopify store connection (shopDomain, accessToken, webhookSecret, registeredWebhooks)
- **checklist_items** - Manual setup steps with completion tracking
- **activity_logs** - System event logging
- **company_settings** - Workspace/company configuration

## API Routes
All routes prefixed with `/api/`:
- `GET/POST /contacts`, `GET/PATCH/DELETE /contacts/:id`
- `GET/POST /conversations`, `GET /conversations/:id/messages`, `POST /conversations/:id/messages` (sends via WhatsApp API)
- `GET/POST /campaigns`, `PATCH/DELETE /campaigns/:id`, `POST /campaigns/:id/send`, `POST /campaigns/:id/send-test`
- `GET/POST /automations`, `PATCH/DELETE /automations/:id`
- `GET/POST /connection`, `POST /connection/test`, `POST /connection/test-message`
- `GET/POST /templates`, `POST /templates/sync`, `DELETE /templates/:id`
- `GET /shopify`, `GET /shopify/auth`, `GET /shopify/callback`, `POST /shopify/disconnect`, `POST /shopify/webhooks`
- `GET/POST /api/webhooks/whatsapp` (Meta webhook verification + incoming messages)
- `GET /checklist`, `PATCH /checklist/:id`
- `GET /activity-logs`
- `GET/POST /settings/company`
- `GET /dashboard/stats`, `GET /dashboard/activity`

## Key Integrations
- **WhatsApp Cloud API**: Real message sending/receiving, template management, credential validation, webhook verification
- **Shopify**: OAuth app installation flow with in-app credential configuration (Client ID + Client Secret stored in DB, not env vars), order/checkout webhooks trigger automations, HMAC signature verification, strict *.myshopify.com domain validation, Shopify Template Builder with Header/Body/Footer/Buttons sections, presets with variable mapping and Quick Reply buttons
- **Automation Engine**: Evaluates active automations on events (order_placed, cart_abandoned, shipping_update, message_received, lead_captured) and sends WhatsApp messages; auto-detects template variables and maps Shopify data
- **Button Reply Handling**: Parses WhatsApp interactive button_reply, list_reply, and template button responses; detects confirm/cancel intents and auto-tags Shopify orders with "Confirmed"/"Cancelled"
- **Phone Normalization**: `normalizePhone()` in storage.ts strips `+`, spaces, hyphens, parentheses from phone numbers. All phone lookups (contacts, conversations, outbound sends) use normalized comparison. Startup migration merges any existing duplicate conversations/contacts created before normalization.
- **WebSocket**: Real-time broadcasts for new messages, status updates, contact creation, campaign completion
